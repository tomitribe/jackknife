= Jackknife

A Maven plugin for inspecting, decompiling, and instrumenting Java jar dependencies.

Jackknife gives you a pocket knife for jars on your classpath — crack them open, read the code, and instrument methods for runtime observation. No source code required.

== Quick Start

Add the plugin to your parent pom (or project pom) with a profile that activates when `.jackknife/` exists:

[source,xml]
----
<profile>
  <id>jackknife</id>
  <activation>
    <file><exists>.jackknife</exists></file>
  </activation>
  <build>
    <plugins>
      <plugin>
        <groupId>org.tomitribe.jackknife</groupId>
        <artifactId>jackknife-maven-plugin</artifactId>
        <version>0.1-SNAPSHOT</version>
        <executions>
          <execution>
            <goals>
              <goal>process</goal>
            </goals>
          </execution>
        </executions>
      </plugin>
    </plugins>
  </build>
</profile>
----

Add the plugin group to `~/.m2/settings.xml` for shorthand invocation:

[source,xml]
----
<pluginGroups>
  <pluginGroup>org.tomitribe.jackknife</pluginGroup>
</pluginGroups>
----

== Usage

=== 1. Index your dependencies

[source,bash]
----
mvn jackknife:index
----

Scans all resolved dependencies, builds structural index files in `.jackknife/index/`, and generates `.jackknife/USAGE.md` with detailed usage instructions.

Index files are flat, greppable text. After indexing, most lookups don't need Maven at all — just grep:

[source,bash]
----
# Find a class
grep -r 'DefaultRetryHandler' .jackknife/index/

# Find implementations of an interface
grep -r 'implements.*HttpRequestRetryHandler' .jackknife/index/

# Find annotated classes
grep -r '@jakarta.ejb.Stateless' .jackknife/index/

# Find methods that throw a specific exception
grep -r 'throws.*IllegalStateException' .jackknife/index/

# Find resources
grep -r 'META-INF/services' .jackknife/index/
----

=== 2. Decompile a class

[source,bash]
----
mvn jackknife:decompile -Dclass=com.example.MyClass
----

Decompiles the class using https://github.com/Vineflower/vineflower[Vineflower] and prints the source to stdout. Results are cached in `.jackknife/source/` for repeat access.

=== 3. Instrument methods

[source,bash]
----
# Instrument a specific method
mvn jackknife:instrument -Dmethod="com.example.Foo.process(String, int)" -Dmode=debug

# Instrument by method name across all classes
mvn jackknife:instrument -Dmethod="retryRequest" -Dmode=timing

# Narrow by interface implementation
mvn jackknife:instrument -Dmethod="retryRequest" -Dimplements=HttpRequestRetryHandler

# Narrow by package
mvn jackknife:instrument -Dmethod="process" -Dpackage=com.example
----

The method parser is forgiving — paste from index output, use Snitch format, or just a method name.

==== Modes

`debug`:: Logs method entry (args), exit (return value), and thrown exceptions. Large values are written to `target/jackknife/captures/` with a file reference in the console.

`timing`:: Logs elapsed time only. Lightweight — no value capture.

=== 4. Build with instrumentation

[source,bash]
----
mvn compile
# or
mvn test
----

The plugin automatically:

1. Picks up instrumentation configs from `.jackknife/instrument/`
2. Transforms matching classes using ASM bytecode modification (rename + InvocationHandler wrapper)
3. Writes patched jars to `.jackknife/modified/`
4. Swaps patched jars into the classpath via `Artifact.setFile()`

No pom changes needed. The original jars in `~/.m2/` are never modified.

=== 5. Revert

[source,bash]
----
# Revert a single dependency
rm .jackknife/modified/org.apache.httpcomponents/httpclient-4.5.13.jar*

# Revert everything
rm -rf .jackknife/modified/*

# Fully remove Jackknife state
rm -rf .jackknife/
----

== How It Works

Jackknife renames instrumented methods (e.g., `process` becomes `jackknife$process`) and generates a wrapper method with the original name that delegates to a `java.lang.reflect.InvocationHandler`. The handler chain is composable:

----
TimingHandler → DebugHandler → ProceedHandler
----

- *TimingHandler* measures elapsed time
- *DebugHandler* logs args, return values, and exceptions
- *ProceedHandler* calls the renamed original method via reflection

Each handler wraps the next. Adding new capabilities is just writing a new `InvocationHandler` — no bytecode work.

== Directory Structure

----
.jackknife/
├── index/                   Greppable index files (one per jar)
│   └── <groupId>/
│       └── <artifact>-<version>.jar.index
├── source/                  Cached decompiled source
├── instrument/              Instrumentation inbox (configs to process)
│   └── <groupId>/
│       └── <artifact>-<version>.jar.properties
├── modified/                Patched jars + receipts
│   └── <groupId>/
│       ├── <artifact>-<version>.jar
│       └── <artifact>-<version>.jar.properties
└── USAGE.md                 Auto-generated usage reference
----

The filesystem is the state machine:

- File in `instrument/` → needs processing
- File in `modified/` → done, actively swapped in
- File deleted → reverted

== Index Format

Each `.index` file is flat text, one class per block:

----
# com.example.MyClass
public class com.example.MyClass extends com.example.AbstractBase
implements java.io.Serializable
@jakarta.inject.Named
  private final field name : java.lang.String
  public method process(java.lang.String input, int count) : boolean
    declares java.io.IOException
    throws java.lang.IllegalArgumentException
  public static method getInstance() : com.example.MyClass

# resources
META-INF/services/com.example.SPI
META-INF/beans.xml
----

Includes access modifiers, generics, parameter names (from debug info), declared exceptions (throws clause), and thrown exceptions (from method bodies).
